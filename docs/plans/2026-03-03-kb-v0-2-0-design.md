# Design: Rosemary KB v0.2.0 вҖ” Multi-file Vector Knowledge Base

**Date:** 2026-03-03
**Status:** Approved
**Supersedes:** `2026-03-03-knowledge-base-design.md`

## 1. Overview

Upgrade Rosemary from a single-topic ingester with LIKE-based search into a
session-aware personal knowledge base that scales to 1M+ records. The core
use-case: at the end of an agent conversation, the session is compacted and
digested into local markdown files and a vector store, so both the user and
future agents can recall it as base knowledge.

### Storage layers

| Layer | Store | Purpose |
|---|---|---|
| Durability | `kb/topics/*.md`, `kb/sessions/*.md` | Human-readable, git-trackable |
| Metadata + Graph | libSQL (FTS5 + relations) | Keyword search, entity graph |
| Semantic | LanceDB | ANN vector search over chunks |

## 3. Data Model

### libSQL

```sql
CREATE TABLE topics (
    id        TEXT PRIMARY KEY,
    title     TEXT NOT NULL,
    file_path TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE VIRTUAL TABLE topics_fts USING fts5(
    title, body,
    content='topics', content_rowid='rowid'
);

CREATE TABLE sessions (
    id        TEXT PRIMARY KEY,
    summary   TEXT NOT NULL,
    file_path TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE entities (
    id          TEXT PRIMARY KEY,
    name        TEXT NOT NULL UNIQUE,
    entity_type TEXT
);

CREATE TABLE relations (
    from_id       TEXT NOT NULL,
    to_id         TEXT NOT NULL,
    relation_type TEXT NOT NULL,
    PRIMARY KEY (from_id, to_id, relation_type),
    FOREIGN KEY (from_id) REFERENCES entities(id),
    FOREIGN KEY (to_id)   REFERENCES entities(id)
);
```

### LanceDB вҖ” `chunks` table

| Field | Type | Notes |
|---|---|---|
| `id` | string | UUID |
| `topic_id` | string | FK вҶ’ `topics.id` or session id |
| `chunk_idx` | uint32 | Position within source file |
| `text` | string | Raw chunk text for display |
| `vector` | vector(384) | Embedding |
| `source` | string | File path |
| `created_at` | timestamp | |

### Chunking strategy

Paragraph-level splitting (`\n\n` boundaries), max 512 tokens per chunk,
64-token overlap between adjacent chunks. Each chunk is independently
embedded and stored as one LanceDB row.

### EmbeddingProvider trait

```rust
#[async_trait]
pub trait EmbeddingProvider: Send + Sync {
    async fn embed(&self, texts: &[String]) -> Result<Vec<Vec<f32>>>;
    fn dim(&self) -> usize;
}
```

Implementations:
- `FastEmbedProvider` вҖ” default, local, `fastembed-rs`, `all-MiniLM-L6-v2` (384-dim)
- `ClaudeProvider` вҖ” remote, Anthropic embeddings API
- `OpenAIProvider` вҖ” remote, `text-embedding-3-small`

Selected via `ROSEMARY_EMBED_PROVIDER` env var (default: `local`).

## 4. Pipelines

### `ingest <file-or-dir>`

1. If directory: walk, collect all `*.md` files
2. For each file:
   - Parse YAML frontmatter вҶ’ title, slug
   - Split body into chunks
   - Batch embed via `EmbeddingProvider`
   - Write chunks вҶ’ LanceDB
   - Upsert topic row вҶ’ libSQL
   - Update FTS5 index

### `digest [--file <transcript>]`

Primary trigger: agent calls at end of session.
Fallback: user runs manually.

1. Read transcript from stdin or `--file`
2. LLM call (model from `ROSEMARY_DIGEST_MODEL`, default `claude-sonnet-4-6`):
   extract distinct topics + a session summary
3. For each topic: write `kb/topics/<slug>.md` (create or append), run ingest
4. Write `kb/sessions/<timestamp>.md` (full session summary), ingest as single chunk

### `recall <query>`

1. Embed query via `EmbeddingProvider`
2. ANN search in LanceDB вҶ’ top-K chunks (default K=20)
3. FTS5 keyword search in libSQL вҶ’ top-N topics (default N=10)
4. Merge + deduplicate by `topic_id`
5. Re-rank: `vector_score * 0.7 + fts5_bm25 * 0.3`
6. Return top-5: chunk text, source file, topic title, related entities

### `relate <from> <to> <type>`

Insert a directed edge into `relations`. Both `from` and `to` must exist in
`entities`; the command creates them if absent.

### `compact [--older-than <days>]`

1. Embed all topic titles, cluster by cosine similarity > 0.85
2. For each near-duplicate cluster: LLM merges into one file, re-ingest,
   delete old rows + LanceDB chunks
3. Prune session files older than `--older-than` (default 90 days) unless
   the topic was recalled in the last 30 days

## 6. Environment Variables

| Variable | Default | Purpose |
|---|---|---|
| `DATABASE_URL` | `rosemary.db` | libSQL file path |
| `KB_ROOT` | `kb` | Root dir for markdown files |
| `LANCEDB_PATH` | `data/lancedb` | LanceDB directory |
| `ROSEMARY_EMBED_PROVIDER` | `local` | `local`, `claude`, `openai` |
| `ROSEMARY_DIGEST_MODEL` | `claude-sonnet-4-6` | LLM for digest + compact |

## 7. Success Criteria

- `digest` at end of a 30-message session produces вүҘ1 topic file and 1 session file
- `recall "rust pinning"` returns the relevant chunk from a past session within 100ms at 100K chunks
- `compact` reduces a set of 5 near-duplicate topic files to 1 without data loss
- Embedding backend is swappable via env var with no code change
